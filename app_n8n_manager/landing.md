# n8n Admin 랜딩 페이지

## 주머니 속 n8n 운영실, 언제 어디서든

워크플로우가 실패했을 때, 노트북을 꺼내지 마세요.
실행 수·성공률·실패 수를 대시보드 한 장으로 확인하고, 문제가 보이면 바로 재시도·중단·전환까지. Cloud와 Self-hosted를 한 앱에서 동시에 관리합니다.

공식 n8n REST API(/api/v1) 위에서 동작하며, 중간 서버 없이 기기와 인스턴스가 직접 통신합니다. API Key·비밀번호는 iOS Keychain / Android Keystore에 안전하게 저장됩니다.

## How it Works

1. 계정 등록 (Cloud / Self-hosted)
   API Key 또는 Email+Password로 n8n 인스턴스를 연결합니다. URL 정규화·연결 테스트·폴백 검증까지 자동으로 수행하며, 시크릿은 기기의 Keychain/Keystore에 분리 저장됩니다.

2. 대시보드에서 상태 3초 진단
   앱을 열면 실행 수·성공률·실패 수 3개 카드가 즉시 뜨고, 그 아래에 워크플로우 그리드가 펼쳐집니다. "오늘 내 자동화는 잘 돌고 있나?"를 3초 안에 답합니다.

3. 실패 실행을 한 탭으로 재시도
   실행 탭 → Error 필터 → 실패 건 선택 → 재시도·중단·삭제. 대시보드 → 실패 실행 → 복구까지 3탭 이내로 끝냅니다.

## Features

1. Insights 요약 대시보드
   총 실행 수, 성공률(%), 실패 수를 3개 카드로 즉시 확인. 태그·활성 여부·보관·오너 필터와 Pull-to-refresh로 현재 상태를 한눈에 파악합니다.

2. 멀티 계정 원탭 전환
   Cloud 인스턴스와 Self-hosted 서버를 한 앱에서 동시에 관리. 좌상단 계정 칩 한 번으로 활성 계정이 전환되고, API 클라이언트가 자동 재바인딩됩니다.

3. 실행 관리 · 재시도 · 중단
   All / Success / Error / Running / Waiting 상태 칩 필터와 무한 스크롤 페이지네이션. 실행 상세에서 재시도·중단·삭제 액션을 바로 수행할 수 있습니다.

4. 노드 그래프 프리뷰
   워크플로우 상세에서 노드 구조 요약(총 노드·트리거·비활성·엣지 수)과 그래프 프리뷰를 확인. 노드 탭 시 해당 노드 하이라이트 + 파라미터 요약이 펼쳐집니다.

5. 보안 우선 저장 구조
   API Key·비밀번호·세션 쿠키는 flutter_secure_storage(iOS Keychain / Android Keystore)에 저장. 계정 메타데이터만 SharedPreferences에 분리 저장하며, 계정별 네임스페이스로 시크릿을 격리합니다.

6. 공식 API 기반의 투명성
   n8n 공식 Public REST API(/api/v1 + X-N8N-API-KEY)만 사용. 중간 서버 없이 사용자 기기가 n8n 인스턴스에 직접 요청합니다. 데이터가 제3자를 거치지 않습니다.

## Who is it for?

- Self-hosted 운영자: VPS·사내 서버에 n8n을 띄워두고 야간·주말에도 관제가 필요한 DevOps를 위해. PC 없이 실패 실행을 바로 확인하고 재시도하세요.
- n8n Cloud 파워유저: 여러 팀·프로젝트 워크플로우를 마케팅·영업·데이터에서 동시 운영. 계정 전환 한 번으로 모든 인사이트를 요약해서 확인합니다.
- 프리랜서·에이전시: 고객사별로 n8n 인스턴스 2~5개를 번갈아 관리하는 환경에 최적화. 멀티 계정 + 빠른 스위칭으로 컨텍스트 전환 비용을 최소화합니다.
- 노코드 빌더·메이커: 개인 자동화 파이프라인을 24/7 모니터링하고 싶은 1인 개발자를 위한 모바일 전용 가벼운 관제 도구.
